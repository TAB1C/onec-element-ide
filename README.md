# onec-element-ide

Skill для Codex, который помогает работать в облачной IDE 1C:Element: открыть workspace по IDE id или URL, внести правки именно в исходники IDE, проверить, что они видны в IDE и Git, закоммитить/синхронизировать изменения, опубликовать приложение и пройти цикл правка-тестирование-исправление.

<опишите задачу здесь>

## Обновление

Тот же текст выше обновит skill, если он уже установлен.

## Что делает skill

- Открывает 1C:Element IDE во встроенном браузере Codex по IDE id или готовому URL.
- Использует авторизованную browser-сессию пользователя; если IDE просит логин, ждет входа пользователя и продолжает в той же вкладке.
- Требует менять исходники именно в IDE workspace, а не только экспортированную сборку или временные файлы.
- Проверяет в IDE search, что новые правки появились, а старый хардкод исчез.
- Перед завершением открывает Git/Source Control в IDE и проверяет список измененных файлов.
- Требует commit и sync/push через IDE Git, чтобы изменения не затерлись следующим pull из Git.
- После сохранения в Git запускает publish/update/test loop и отделяет проблемы 1C-обертки от проблем внешнего backend.
