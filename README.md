# onec-element-ide

Skill для Codex, который помогает работать с проектами 1C:Element через панель управления: скачать исходники локально, внести правки в рабочей папке, загрузить исправленные файлы обратно в панель, проверить видимость изменений в IDE/Git, опубликовать приложение и пройти цикл правка-тестирование-исправление.

## Запуск одним сообщением

Вставьте в чат Cursor/Codex этот текст и допишите задачу внизу:

```text
Подтяни или обнови репозиторий https://github.com/TAB1C/onec-element-ide в ~/.codex/skills/onec-element-ide, прочитай ~/.codex/skills/onec-element-ide/SKILL.md и дальше работай по нему для IDE <IDE id или URL>. Запроси у меня ключ и секрет панели управления 1C:Element, если я не передал их сразу.


Задача:
<опишите задачу здесь>
```

## Обновление

Тот же текст выше обновит skill, если он уже установлен.

## Что делает skill

- Запрашивает ключ и секрет панели управления 1C:Element, если они не переданы сразу.
- Скачивает исходники или экспорт проекта локально через панель управления/API.
- Помогает править `.xbsl`/`.yaml` файлы в локальной рабочей папке и проверять хардкод локальным поиском.
- Загружает исправленные файлы или пакет обратно в панель управления.
- Проверяет через IDE/панель, что изменения действительно видны в облачном source view.
- Проверяет Git/Source Control, commit/sync/push, если проект показывает это в IDE или панели управления.
- После загрузки запускает publish/update/test loop и отделяет проблемы 1C-обертки от проблем внешнего backend.
